A² Accessories

This production-ready app is a wedding accessory store in the same style as the site I made for Glam by Adelina. It is able to process payments and will be used for business once I add a cart and shipping feature.

Features:

  • Forms and automatic mailers
  • Authentication and authorization
  • Validation and pagination
  • Sass and CoffeeScript for preprocessing bliss
  • Unit and functional tests using RSpec
  • Caching with AJAX and Redis
  • ActiveJob and ActionCable for streaming and broadcasting of ratings and comments
Link
Code

Only admin accounts should be able to edit or destroy products, so I used Ruby not only to hide the CRUD interface from non-admins, but to prevent any unauthorized commands through the URL (e.g. "/product/1/edit"). I also tested for extreme cases (such as a user typing in a non-existent product ID) and made sure to "rescue" wayward users by adding redirects to my config file.

The RSpec tests I wrote ensured consistent success for user submitted comments, ratings, and the validation and math underlying them. One of the most challenging (and heavily test-dependent) features to develop involved using ActionCable to update the page live every time a new user submits a comment.

I used various forms of data storage and retrieval such as PostgreSQL, Memcached, and Redis to make the site run as quickly and securely as possible, but I needed to rely on a third party for credit card payments. This decision was principally for legal and security reasons, namely that I would be too much at risk to hold credit card information on my server. I implemented Stripe and used the full flexibility of their API to match it to our brand and route to a custom email once visitors made a purchase.
